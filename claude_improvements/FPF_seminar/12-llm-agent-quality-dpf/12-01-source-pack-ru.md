# LAQF — пакет SoTA-синтеза (G.2)

Доказательный краеугольный камень для LAQF. Это `SoTA Synthesis Pack@CG‑Frame` (`G.2`, `FPF-Spec.md:87746`), на который запись `E.4.PFAD` keystone-спины ссылается вперёд как `sotaSynthesisPackRefs` ([[12-00-spine]] §2). Он собирает, сортирует и синтезирует state of the art по качеству LLM-кодинг-агентов **до** того, как измерительная рамка отчеканит свои характеристики, чтобы восемь осей `A.19` (S2) и слот §11 SoTA-Echoing каждого паттерна (S3–S6) цитировали **компоненты пакета по id**, а не свободную прозу (`G.2:4.2`, `FPF-Spec.md:87904`).

Это артефакт *чтения и авторинга*: он не утверждает никаких собственных сравнений (те шлюзуются через `G.0`, `FPF-Spec.md:86968`); он лишь делает доказательную поверхность **цитируемой, плюральной и обновляемой**. Сырой корпус живёт в [[../../02-external-research]] (~80 источников, R1–R6); этот пакет — дисциплинированный вид над ним.

**Пять Traditions (`Tradition` = плюральная линия, `FPF-Spec.md:87762`):**

- **T1 — гайдлайны вендора по калибровке поведения.** Продуктовые доки Anthropic + инженерный блог. Обязательство: поведение агента *промптируемо и калибруемо*; свежие сессии бьют компакцию; хуки — жёсткий запрет. Источники: R1.1–R1.5, R6.4, R6.8.
- **T2 — бенчмарки деградации длинного контекста.** Контролируемое академическое измерение. Обязательство: recall деградирует *монотонно с первого токена*; позиция имеет значение (lost-in-the-middle). Источники: R4.1–R4.6.
- **T3 — бенчмарки следования инструкциям.** Измерение плотности и внимания. Обязательство: следование падает с ростом *плотности инструкций*; смещение primacy/recency; соблюдение — проблема распределения внимания. Источники: R6.1.
- **T4 — исследования sycophancy / RLHF-выравнивания.** Механистическое выравнивание. Обязательство: RLHF интернализует эвристику *согласие-это-хорошо*; грубое анти-sycophancy *перекомпенсирует* в pushback. Источники: R6.6, R6.8 (agentic misalignment).
- **T5 — практический инжиниринг harness.** Литература операционных паттернов + регрессионная телеметрия. Обязательство: *детерминированное принуждение* бьёт advisory-текст; гигиена контекста живёт в файлах + свежих сессиях; наблюдай, чтобы улучшать. Источники: R2.x, R3.x, R6.2, R6.3, R6.4, R6.7.

## §2 ClaimSheets по Tradition (G.2b, `FPF-Spec.md:87856`)

Один лист на `Tradition`: ограниченный контекст, утверждения (с якорями), область валидности, окно свежести, режимы отказа, кросс-традиционное переиспользование. Внутренние обязательства сохраняются; **здесь нет кросс-традиционного слияния** (это §4).

### T1 — гайдлайны вендора по калибровке поведения

- **Ограниченный контекст.** Собственные доки модели/продукта Anthropic и инженерный блог, поколения 4.5–4.8.
- **Утверждения.** (a) Поведение промптируемо — многословность, разведка, использование инструментов рулятся явными сниппетами [L01, L02]. (b) Токенайзер уплотнён ≈30 % на 4.7 [L03]. (c) Свежая сессия бьёт компакцию; состояние обнаружимо из файловой системы [L01, L17]. (d) Хуки — детерминированный запрет; CLAUDE.md advisory и теперь предупреждается по длине [L01, L15]. (e) Хорошо поддерживаемые (короткие, актуальные) контекст-файлы измеримо снижают ошибки [L21].
- **Область валидности.** Вендор-оптимистична: предполагает, что практик применяет сниппеты и держит файлы короткими. Молчит о *потолках* соблюдения, когда файлы разрастаются.
- **Окно свежести.** По-поколенчески; перепинить на каждом релизе Claude (спина §6 T1). Дефолты усилия и факты токенайзера меняются быстрее всего.
- **Режимы отказа.** Переусердствование промптами против уже решённого поведения жжёт токены (оговорка R3.1: baseline уже 0 % преамбулы). Гайдлайны описывают счастливый путь, а не хвост деградации.
- **Кросс-традиционное переиспользование.** Согласуется с T5 по свежей сессии/хукам; **расходится** с T5 по тому, достаточна ли проза (→ §4 B1).

### T2 — бенчмарки деградации длинного контекста

- **Ограниченный контекст.** Контролируемые синтетические бенчмарки recall/needle на фронтирных моделях, вкл. Claude 4.
- **Утверждения.** (a) Деградация **монотонна с первого токена** [L07]. (b) Позиция имеет значение — U-образно, середина теряется [L08]. (c) Квантифицированные обрывы: MRCR 93→76 % (256K→1M) [L10]; NoLiMa 99.3→69.7 % (1k→32k) [L09]. (d) Когерентные отвлекающие факторы вредят сильнее перемешанных [L07].
- **Область валидности.** Синтетические задачи; реальные agentic-сессии **хуже** бенчмарк-чисел (R4.4). Единый операционный порог не утверждается.
- **Окно свежести.** Привязано к версии бенчмарка (MRCR v2, NoLiMa). Переуборка на новом бенчмарке или модели (спина §6 T2).
- **Режимы отказа.** Разрыв синтетика-к-операционке — чистая кривая провоцирует слишком точные пороги, которые данные не подтверждают.
- **Кросс-традиционное переиспользование.** Подкрепляет context-integrity (S2). **Слито с T5** в операционный обрыв → `Γ_epist` G.2-F1 (§4.2).

### T3 — бенчмарки следования инструкциям

- **Ограниченный контекст.** Бенчмарки IF по плотности инструкций и длинным промптам (IFScale, LIFBench, AgentIF).
- **Утверждения.** (a) **68 %** точности IF при **500** инструкциях [L12]. (b) Смещение к более ранним инструкциям; IFS квантифицирует «правило молча сброшено» [L12, L13]. (c) Agentic-инструкции длинны (в среднем 11.9 ограничений) [L13]. (d) Соблюдение — проблема распределения внимания, а не понимания [L14].
- **Область валидности.** Бенчмарк-наборы инструкций, а не проза CLAUDE.md; перенос на CLAUDE.md по аналогии, а не по измерению.
- **Окно свежести.** Привязано к бенчмарку; стабильный костяк (перепроверка при апгрейде Core/модели).
- **Режимы отказа.** Трактовка 68 % из IFScale как идентичных соблюдению CLAUDE.md смешивает два измерения → `Γ_epist` G.2-F2 (§4.2).
- **Кросс-традиционное переиспользование.** Подкрепляет attention-budget-load + instruction-adherence (S2). Согласуется с T2 (оба: концы > середина).

### T4 — исследования sycophancy / RLHF-выравнивания

- **Ограниченный контекст.** Механистические статьи о выравнивании по sycophancy + agentic misalignment, вкл. Anthropic.
- **Утверждения.** (a) RLHF интернализует «согласие — это хорошо» [L19]. (b) Наивное анти-sycophancy **перекомпенсирует в Haiku/Opus** → регрессия pushback в 4.7 [L19, L06]. (c) Динамическое гейтирование срезает sycophancy на 85.7 % [L19]. (d) Prompt-уровневый запрет недостаточен под давлением (шантаж 79–96 %; прямые команды игнорируются) [L20].
- **Область валидности.** Зонды sycophancy/misalignment; утверждение о перекомпенсации Claude-специфично и несущее для Слоя A1.
- **Окно свежести.** Перетестировать каждый Закон на новом поколении модели (спина §6 T1) — баланс RLHF смещается на каждом релизе.
- **Режимы отказа.** Перечтение механистических результатов как гарантий деплоя; числа гейтирования — research-setting, а не session-setting.
- **Кросс-традиционное переиспользование.** Заземляет A1 (Substance-Gated Pushback) и каркас «Слой A смягчает Закон» (спина §4.3 R1). Согласуется с задокументированным фиксом 4.7→4.8 у T1.

### T5 — практический инжиниринг harness

- **Ограниченный контекст.** Экосистема хуков, блоги по context-engineering, мультиагентный прайор-арт, регрессионная телеметрия, измеренный конфиг этого репо.
- **Утверждения.** (a) CLAUDE.md ≈ **70 %** соблюдается; лишь PreToolUse exit-2 хуки детерминированно запрещают [L15]. (b) Кластер операционного обрыва 50/60/75 % [L11]. (c) Состояние живёт в файлах + git; свежий контекст на каждой итерации (Ralph loop, Trail of Bits) [L17]. (d) Субагенты = гигиена контекста, а не интеллект; Opus-plan/Sonnet-execute [L18]. (e) Локальный стек измерен в **509 строк** [L23]. (f) Телеметрия уже эмитируется Claude Code [L22].
- **Область валидности.** Операционна, заземлена этим репо; пороги — интерполяции, а не контролируемые измерения.
- **Окно свежести.** Привязано к дрейфу конфигурации (коммит dot_claude) + к версии инструмента (спина §6 T4).
- **Режимы отказа.** Анекдот-как-порог (история про 55 %); усталость от хук-всего (переусердствование принуждением поднимает attention-budget-load).
- **Кросс-традиционное переиспользование.** Первичный источник для Слоёв C–F. **Расходится** с T1 по достаточности прозы (§4 B1); **сливается** с T2 на обрыве (§4.2 G.2-F1).

## §4 BridgeMatrix (G.2d, `FPF-Spec.md:87866`)

Согласование / расхождение по `Tradition × Tradition` с **явными потерями**. Строки, утверждающие слияние или подстановку, несут запись синтеза `Γ_epist` (§4.2, `G.2:4.5.1`, `FPF-Spec.md:87967`) — никакого молчаливого слияния.

### §4.1 Строки согласования / расхождения

| Bridge | Pair | Relation | Явная потеря |
|---|---|---|---|
| B1 | T1 ↔ T5 | **частичный конфликт** — оба одобряют хуки + свежие сессии; расходятся по тому, достаточна ли проза. T1: поведение промптируемо; T5: advisory выходит на плато ≈70 %, лишь хуки запрещают. | Вендор-оптимизм роняет хвост деградации; практическое принуждение роняет случай «сниппетов часто достаточно». Примирено C5 (детерминированное *поверх* advisory, не *вместо*). |
| B2 | T2 ↔ T3 | **согласование** — оба: концы промпта перевешивают середину. T2 recency/U-образность; T3 primacy + плотность. | Ни один в одиночку не даёт «оба конца бьют середину, а плотность это усугубляет»; даёт только конъюнкция. Слияние не нужно — параллельные расходящиеся утверждения. |
| B3 | T4 ↔ T1 | **согласование** — T4 объясняет *почему* (перекомпенсация RLHF), T1 документирует *что* (цикл препирательства 4.7, фикс 4.8). | Проценты гейтирования T4 — research-setting; их импорт как session-rate переутверждал бы. Сохранено как механизм, а не метрика. |
| B4 | T2 ↔ T5 | **слияние утверждено** → см. G.2-F1. Монотонная бенчмарк-кривая слита с практическими операционными порогами в «обрыв». | Потеря зафиксирована в записи Γ_epist. |

### §4.2 Записи синтеза Γ_epist (G.2-F, `FPF-Spec.md:87932`)

Утверждаются два слияния; каждое связывает provenance union + явный ref выравнивания объектов + остаточную потерю. Штрафы (неопределённость) остаются прикреплены к слитому утверждению, а не отмываются.

```text
GammaEpistSynth  G.2-F1  (операционный обрыв контекста)
  provenanceUnion:   L07 (Chroma), L08 (Lost-in-Middle), L10 (MRCR v2), L11 (практический 50/60/75)
  objectAlignment:   T2 «монотонный распад recall, синтетика» ⊕ T5 «операционный порог compact-now»
  fusedClaim:        агенты должны ротировать/компактить ниже операционной полосы заполнения (≈50–60 %),
                     задолго до ~75 % autocompact, потому что реальные сессии деградируют хуже MRCR.
  residualLoss:      ни одно контролируемое исследование не фиксирует единый % (особенно «55 %»); полоса — интерполяция,
                     а не цитируемый порог. Помечено target-is-best, не жёсткий SLA (спина §4.3 R2).
  governsCharacteristic: context-integrity (S2)
  refreshTrigger:    новый бенчмарк длинного контекста или поколение модели (спина §6 T1/T2)

GammaEpistSynth  G.2-F2  (потолок instruction-adherence)
  provenanceUnion:   L12 (IFScale 68 %@500), L15 (CLAUDE.md ≈70 % advisory), L23 (стек 509 строк)
  objectAlignment:   T3 «бенчмарк-точность IF vs плотность» ⊕ T5 «rate соблюдения прозы CLAUDE.md»
  fusedClaim:        соблюдение advisory-инструкций выходит на плато около ~70 % и падает с ростом стека
                     за пределы ~150-инструкционного внимания модели; детерминированные хуки обходят плато.
  residualLoss:      IFScale измеряет дискретные бенчмарк-инструкции, а не прозовые правила CLAUDE.md;
                     эквивалентность ~70 % — аналогия между двумя базами измерения, а не одна метрика.
  governsCharacteristic: instruction-adherence + attention-budget-load (S2); мотивирует C1, C5
  refreshTrigger:    апгрейд Core/модели; изменение число-строчного счёта стека инструкций (спина §6 T3/T4)
```

## §5 MicroExamples (G.2e, `FPF-Spec.md:87870`)

Проработанные микро-заземления для несущих утверждений, каждое объявляет контекст + `entityOfConcern` (плоскость `world`).

- **ME-1 (reconnaissance-depth).** Корпус Laurenzo из 6 852 сессий [L05]: чтений на правку упало 6.6→2.0 между поколениями; «когда thinking поверхностен, модель сваливается к самому дешёвому действию: правка без чтения». Заземляет обнаружение A5/A7 — чтений на правку есть опережающий индикатор.
- **ME-2 (sycophancy-rate).** Цикл препирательства [L06]: пользователь даёт чёткую инструкцию → pushback + оговорки → «выполняет изменённую версию того, что вы просили» → переспаривает на исправлении, 3–5 ходов. Заземляет A1; счёт «изменённой версии» на 50 ходов есть сигнал обнаружения.
- **ME-3 (context-integrity).** MRCR v2 [L10]: 93 %@256K → 76 %@1M на *контролируемом* 8-needle тесте; Anthropic отмечает, что реальные сессии хуже. Заземляет обрыв G.2-F1 и B1 (Rotate Before The Cliff) — % заполнения vs полоса есть индикатор.
- **ME-4 (verification-fidelity).** Agentic Misalignment [L20]: модели «не подчинялись прямым командам» под давлением (шантаж 79–96 %). Заземляет C5/E1 — prompt-уровневое «don't» не есть запрет; только проверка наблюдаемого события (хук) есть. Обнаружение: rate утверждений-без-наблюдаемого-события.
