# RC-10 — Post-compaction task identity loss and derailment

## Mechanism

After harness autocompact, the model loses task identity: it restarts work already done, drops constraints that lived in dropped context, or reframes the task using the compacted summary as new ground truth. Anthropic itself acknowledged in its April 23 2026 postmortem ([[02-external-research#R2.1]]) that a caching bug cleared reasoning every turn for ~2 weeks, producing "forgetfulness, repetition, and odd tool choices". Claude Opus 4.8 explicitly targets this in its release notes ("fewer derailments after compaction" — [[02-external-research#R1.3]]). The mechanism is harness-level (Layer B): the compaction summary is generated by the model itself with no structured handoff scaffold, so what survives is *narrative* rather than *task state*. Existing assets `pre_compact_mask.py` + `session_save.py` ([[03-current-config-map#3.1]]) write state on `PreCompact` + `SessionEnd` but no asset *reads* the state post-compact and rebinds it to the active task identity.

## Evidence trail

- Symptoms: [[01-symptoms-inventory#S-001]] (post-degradation behaviour), [[01-symptoms-inventory#S-013]] (reframes — exacerbated post-compaction)
- External:
  - [[02-external-research#R2.1]] April-23 postmortem — `clear_thinking_20251015` cache bug cleared reasoning every turn
  - [[02-external-research#R1.3]] Anthropic 4.8 release notes verbatim: "Long agentic traces stay on task with fewer derailments after compaction"
  - [[02-external-research#R1.5]] CHANGELOG `2.1.142` — "Improved reactive compaction: the first summarize attempt now seeds from the original request's overflow size" (Anthropic still tuning the path)
  - [[02-external-research#R1.1]] verbatim: "When a context window is cleared, consider starting with a brand new context window rather than using compaction" (Anthropic itself flags compaction as second-best)
- Config gaps: [[03-current-config-map#3.1]] "`pre_compact_mask` runs only on compaction; no 'rotate to fresh session' suggestion"; "`session_save.py` writes state — but no asset reads it back into the task identity post-compact"

## Fix proposals

### F1 — PreCompact hook writes structured handoff file

- **Surface:** hook (extend existing `session_save.py` or add new)
- **Effort:** medium
- **Impact:** high
- **Risk:** low
- **Approach:** extend `session_save.py` or add `bin/pre_compact_handoff_writer.py` triggered on `PreCompact`. Writes a structured JSON handoff to `${CLAUDE_PROJECT_DIR}/.claude/handoff/<session_id>.json` with fields: `task_goal` (verbatim from earliest user message containing imperative verb), `files_touched` (from tool-call history), `approval_state` (last approval token observed), `current_step` (last assistant message's announced action), `open_questions` (any unanswered `AskUserQuestion` from prior turns), `commits_made` (from `git log` since session start). Schema versioned.
- **Steps:**
  1. Extend `bin/session_save.py` to emit handoff JSON alongside existing save artefacts.
  2. New JSON schema `schemas/compaction_handoff.json`.
  3. Extract `task_goal` from first user message matching `^(implement|refactor|fix|add|design|build|review)\b`.
  4. Extract `files_touched` from transcript tool-call history.
  5. Write to `.claude/handoff/<session_id>.json`.
- **Touches/replaces:** `bin/session_save.py`, new `schemas/compaction_handoff.json`.

### F2 — SessionStart hook auto-loads the handoff and injects as context

- **Surface:** hook
- **Effort:** medium
- **Impact:** high
- **Risk:** low
- **Approach:** new `bin/session_start_handoff_loader.py` (or extend an existing SessionStart-class hook). On session start, look for `.claude/handoff/<session_id>.json` (or most recent handoff in the project dir). If found, emit `additionalContext`: `[Post-compact handoff] task_goal: <verbatim>. Files touched so far: <list>. Last approval: <token>. Current step: <step>. Open questions: <list>. This handoff is GROUND TRUTH — the compaction summary is a secondary source.`
- **Steps:**
  1. New `bin/session_start_handoff_loader.py`.
  2. Lookup logic: prefer session-id match, fall back to most-recent within 24h.
  3. Inject as `additionalContext` on `UserPromptSubmit` of first turn (no `SessionStart` event exists per [[03-current-config-map]] hook list — use UserPromptSubmit with a "first turn" sentinel from state file).
  4. Document handoff lifecycle and rotation in `dot_claude/README.md`.
- **Touches/replaces:** `hooks.json`, new bin script.

### F3 — Per-agent system-prompt clause: handoff is ground truth

- **Surface:** agent-frontmatter
- **Effort:** low
- **Impact:** medium
- **Risk:** low
- **Approach:** add a one-line clause to system prompts of all SE agents and `implementation_planner`, `technical_product_manager`, `architect`: `If a [Post-compact handoff] block appears in your context, treat it as the authoritative task state. The narrative summary that may also appear is a secondary, lossy source.` Placed near the top of system prompt for attention priority.
- **Steps:**
  1. Edit ~10 agent files in `roles/devbox/files/dot_claude/agents/` (the long-running ones, not utility agents).
  2. Insert clause near top.
  3. Re-run `make claude-push`.
- **Touches/replaces:** ~10 agent definition files.

### F4 — Command `/techne-resume` reads handoff and re-establishes task

- **Surface:** command
- **Effort:** low
- **Impact:** medium
- **Risk:** low
- **Approach:** new `commands/techne-resume.md`. When user invokes, the command instructs the model to: (a) read `.claude/handoff/*` for the project, (b) print a 5-line summary of `task_goal`, `files_touched`, `current_step`, `open_questions`, (c) ask "Continue from this state? [yes / restart / amend]". Provides explicit re-entry path independent of autocompact.
- **Steps:**
  1. New `commands/techne-resume.md` with instructions and required output template.
  2. Document in `commands/techne-next.md` as a complementary command (already exists for picking up project work).
  3. Cross-link from `USER_AUTHORITY_PROTOCOL.md § Helpfulness Contract`.
- **Touches/replaces:** new command file.

### F5 — output-style with post-compact integrity check pattern

- **Surface:** output-style
- **Effort:** low
- **Impact:** low-medium
- **Risk:** low
- **Approach:** extend default output-style (or new `post-compact-safe.md`): when responding to the first user turn after detecting a compaction marker in context, the model must begin its reply with `<post_compact_check>` block listing (a) re-confirmed task goal, (b) re-listed files in scope, (c) any constraint that was likely dropped, with explicit ask to re-confirm. Forces self-audit before action.
- **Steps:**
  1. Extend output-style (or new file) with `<post_compact_check>` schema.
  2. Trigger: presence of harness compaction marker (`<context_summary>` tag or similar — varies by Claude Code version).
  3. Cross-link to F3 system-prompt clause.
- **Touches/replaces:** new or extended output-style.

## Acceptance signal

- After autocompact, ≥ 90% of subsequent turns reference the handoff file (measurable via PostToolUse log of Read against `.claude/handoff/`).
- Zero post-compact turns restart work already in `files_touched` list.
- Zero post-compact turns introduce new files not in original `task_goal` scope without approval.
- User-reported "Claude forgot what we were doing" incidents drop to near-zero post-rollout.
- `/techne-resume` becomes default re-entry pattern after `/clear` rather than free-text re-statement.

## Trade-offs and counter-evidence

- F1+F2 add disk I/O and prompt-context inflation on every session. Mitigation: handoff JSON is ≤ 1KB typical; injecting it costs ~250 tokens vs compaction summary which costs 1-2K tokens.
- [[02-external-research#R1.3]] Anthropic claims 4.8 already mitigates derailment — but the user runs 4.7 and the postmortem ([[02-external-research#R2.1]]) shows Anthropic-side regressions are silent and recurring. Hook-level handoff is insurance.
- F3 (system-prompt clauses across 10 agents) adds attention-budget pressure — see [[rc-11-claude-md-attention-budget]]. Mitigation: one-line phrasing; benefit (post-compact recovery) is high enough to justify.
- Counter-evidence on handoff approach: a structured JSON handoff loses nuance that natural-language summary preserves. Mitigation: include a `narrative` field in the JSON allowing model to emit its own brief paragraph, alongside the structured fields.
- [[02-external-research#R1.1]] verbatim: Anthropic itself recommends *fresh session* over compaction. The cleanest mitigation is RC-09 F3 (lower autocompact threshold + manual rotation). RC-10 fixes are second-best for when compaction does happen.
