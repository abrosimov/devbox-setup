# Выбор имени проекта — независимый прогон F.18

Итерация 0 ([[choosing_name]]) выбрала **Κάτοπτρον**, опираясь на три принципа-суждения
(A.7 Strict Distinction, A.6.B Laws vs Work, A.15 Work). Этот файл заново прогоняет
вопрос об имени через **F.18 — Local-First Unification Naming Protocol** как
*независимое* упражнение: протокол применяется без предположения о прежнем ответе,
поэтому результат волен подтвердить Κάτοπτρον, уточнить его или заменить.

F.18 — другой инструмент, нежели принципы итерации 0. Там, где A.7/A.6.B/A.15 спрашивают
*«что это за вещь и что её имя вправе утверждать?»*, F.18 спрашивает *«какова устойчивая,
переиспользуемая, проверенная на коллизии метка для уже-управляемого значения, выбранная
из видимого набора кандидатов по объявленным ординарным измерениям, с зафиксированной
родословной?»* (`FPF-Spec.md:85840`, `:85914`).

> **Замечание по спеке.** В семинаре протокол назван «F18 / F.48». В спецификации FPF
> есть **F.18 — Local-First Unification Naming Protocol** (`FPF-Spec.md:344`, `:85840`);
> F.48 не существует (серия F заканчивается на F.19). Имя файла сохраняет дословно
> запрошенный `_48`; применяемый *метод* — F.18.

## Шаг 1 — Восстановить governed value (F.18:0, F.18:4 шаг 1)

Первый ход F.18 — восстановить управляемое значение и его управляющий паттерн *до*
выбора метки, чтобы имя не протащило в себя онтологию (`FPF-Spec.md:85853`).

| Вопрос (инварианты F.18:4.1) | Ответ для этого проекта |
|---|---|
| **Governed value first** | Проект `claude_improvements/`: текущая **диагностико-митигирующая Work** (`A.15`), которая порождает и потребляет **рефлексивный корпус**, характеризующий поведенческое пространство Claude Code (`A.19` CharacteristicSpace). 36 файлов, ~4900 строк, 41 симптом → 30 корневых причин → 150+ предложений по исправлению ([[../00-MoC]]). |
| **Governing pattern visible** | `A.15` (Work) для деятельности; `A.19` (CharacteristicSpace) для корпуса; сама устойчивая метка выбирается по `F.18`. |
| **Bounded context visible** | Рабочее пространство `AION_AUTOPOIESEON`, контекст проекта `claude_improvements`. Не Core-facing, не cross-context — поэтому публикация публичной term-row через `F.17` **не** задействована (F.18:4.4). |
| **Local sense visible** | «Рефлексивный инструмент самоосмотра поведения Claude Code: симптомы — искажения в зеркале, корневые причины — оптика зеркала, FPF-семинар проверяет кривизну самого зеркала». |

**Двойственность вида значения (зафиксирована явно).** Управляемое значение читается двояко:

- **Прочтение «инструмент»** — проект *есть* рефлексивная поверхность / корпус. Корпус
  ссылается на себя именно так («каталог Katoptron», «зеркало») в [[11-fpf-diagnostic]] и
  [[../00-MoC]]. Существительное-инструмент подходит.
- **Прочтение «Work»** — проект *есть* текущая деятельность (диагностировать,
  предписывать, аудировать). Имя в форме деятельности/метода подходит лучше.

Этот выбор прочтения задаёт измерение **MorphologicalActionFit** ниже, поэтому объявлен
заранее, а не спрятан внутри скоринга. Итерация 0 остановилась на прочтении *инструмент*
(пункт 1 её обоснования: «Katoptron именует *ту вещь, которой проект является*, —
рефлексивную поверхность»). Этот прогон принимает то же основное прочтение **и** несёт
прочтение «Work» как зафиксированный остаточный риск (см. обоснование выбора и
RefreshCondition).

## Шаг 2 — Решение Mint / Reuse / DocumentLegacy (F.8 через F.18:4)

**MintNew** (умолчание F.18, `FPF-Spec.md:13640`). Внешне зафиксированной стандартной
метки нет; `DocumentLegacy` нечего применять. Κάτοπτρον из итерации 0 — внутренний прежний
выбор, а не внешний стандарт, поэтому этот прогон волен перечеканить имя заново.

## Шаг 3 — Набор кандидатов (F.18:4.3)

F.18 требует 5–10 кандидатов из **минимум двух семейств головных терминов**, оцениваемых
по ординарным измерениям, и **никогда не усреднять их в один балл**
(`FPF-Spec.md:85974`–`:85981`).

### Семейство A — Оптика / отражение
- **A1 · Κάτοπτρον (Katoptron)** — зеркало; рефлексивный *инструмент*. κατά + ὄπτομαι = «смотреть на себя».
- **A2 · Κατοπτρισμός (Katoptrismos)** — отражение / зеркалирование *как процесс* (суффикс деятельности -ισμός). Также обыденно-греческое значение «мираж».
- **A3 · Δίοπτρα (Dioptra)** — *прицельный / измерительный* инструмент (смотреть-сквозь, а не смотреть-на-себя).

### Семейство B — Диагностика / медицина
- **B1 · Ἀνάμνησις (Anamnesis)** — история болезни / припоминание.
- **B2 · Διάγνωσις (Diagnosis)** — различение-насквозь; именует диагностический *акт*.

### Семейство C — Диалектика / испытание
- **C1 · Ἔλεγχος (Elenchus)** — сократический перекрёстный допрос; именует *деятельность/метод* испытания.

### Семейство D — Парадокс
- **D1 · Φάρμακον (Pharmakon)** — лекарство, которое одновременно яд.

## Шаг 4 — Ординарный скоринг и NQD-front (F.18:4.3)

Измерения — собственный лексический вектор качества F.18 (`FPF-Spec.md:25024`,
`:85976`–`:85979`): **SemanticFidelity** (сохраняет управляемое значение, не добавляя и не
теряя условий), **CognitiveErgonomics** (читатель может произнести и запомнить),
**MorphologicalActionFit** (форма слова подходит виду именуемого), **AliasRisk** (импортирует
ли внимательный читатель ошибочный смысл — *меньше = лучше*).

Шкала ординарная: `High / Med / Low`. Значения **не** усредняются.

| # | Кандидат | SemanticFidelity | CognitiveErgonomics | MorphologicalActionFit | AliasRisk *(low = good)* |
|---|---|---|---|---|---|
| A1 | Κάτοπτρον | **High** — зеркало = самоосмотр; «катоптрика» (наука об отражении) несёт расширяемость | **High** | **High** для прочтения *инструмент* | **Low** — «катоптрический» достаточно редок, чтобы не тащить ложный прототип |
| A2 | Κατοπτρισμός | High — отражение-как-процесс | Med — длиннее, менее чёткое | High для прочтения *Work* | **High** — «мираж» коннотирует *иллюзию / нереальность*, что подрывает претензию диагностики на истину |
| A3 | Δίοπτρα | Med — смотреть сквозь, а не отражать себя | Med | Med | Med — прототип геодезического прибора |
| B1 | Ἀνάμνησις | Med — только память; проект шире | Med | Med | High — медицинский + платонический багаж |
| B2 | Διάγνωσις | Med — именует лишь диагностическую половину; проект ещё и предписывает и аудирует | High — интернационально знакомо | High для прочтения *Work* | High — чисто клинический прототип |
| C1 | Ἔλεγχος | Med — именует *метод*, не объект | Med | High для прочтения *Work* | Med — конфронтационный тон может быть перечитан |
| D1 | Φάρμακον | Med — именует *напряжение*, не проект | Med | Low — именует *вещество*, не работу и не инструмент | High — позднее значение «колдовство / яд» |

**Правило доминирования.** Кандидат *X* доминирует *Y* ⟺ *X* не хуже *Y* по всем четырём
измерениям и строго лучше хотя бы по одному (AliasRisk инвертирован). **NQD-front** —
множество недоминируемых кандидатов.

**Front при основном прочтении (инструмент):**

- **A1 Κάτοπτρον** — недоминируем. При этом прочтении его никто не превосходит ни по
  одному измерению.
- A2, A3, B1, B2, C1, D1 — каждый доминируется A1: A1 не хуже их по всем измерениям.
  (Напр. A1 vs A2: равная SemanticFidelity, A1 выигрывает CognitiveErgonomics и AliasRisk,
  равно-или-лучше MorphologicalActionFit → A1 доминирует A2. A1 vs C1: A1 выигрывает
  SemanticFidelity, CognitiveErgonomics и AliasRisk, по MorphologicalActionFit равенство
  лишь при прочтении *Work* → A1 доминирует C1 при прочтении «инструмент».)

→ **Front = { Κάτοπτρον }** (легитимный синглтон: один кандидат доминирует).

**Front при альтернативном прочтении (Work)** — зафиксирован, не выбран:

- Если управляемое значение привилегировать как *текущую Work*, а не как *инструмент*,
  колонка MorphologicalActionFit переворачивается для имён-деятельностей, и front
  раскрывается до **{ Κάτοπτρον, Ἔλεγχος, Κατοπτρισμός }** — Ἔλεγχος и Κατοπτρισμός тогда
  обходят Κάτοπτρον по MorphologicalActionFit, проигрывая по AliasRisk, поэтому никто не
  доминирует.

## Шаг 5 — Выбор и Name Card (F.18:4.2)

Выбран **Κάτοπτρον**, с заявленным остаточным риском по MorphologicalActionFit, как
требует F.18 («один кандидат может выиграть, даже будучи неидеальным, но SelectionRationale
обязан сказать, что он покупает и какой риск остаётся», `FPF-Spec.md:85983`).

```text
NameCard:
  NameCardId:        katoptron-f18-iter2
  GovernedValueRef:  проект claude_improvements — рефлексивная диагностическая Work (A.15),
                     порождающая поведенческий корпус-CharacteristicSpace (A.19)
  GoverningPatternRef: A.15 (Work) + A.19 (CharacteristicSpace); метка по F.18
  BoundedContextRef: AION_AUTOPOIESEON / claude_improvements
  LocalSenseRef:     «рефлексивный инструмент самоосмотра поведения Claude Code»
  TechLabel:         Κάτοπτρον (Katoptron)
  PlainLabel:        каталог Katoptron / зеркало
  CandidateSet:      A1 Κάτοπτρον; A2 Κατοπτρισμός; A3 Δίοπτρα;
                     B1 Ἀνάμνησις; B2 Διάγνωσις; C1 Ἔλεγχος; D1 Φάρμακον
  RejectedCandidates:
    - Κατοπτρισμός: AliasRisk High — «мираж» коннотирует иллюзию, разъедает претензию
      диагностики на истину; доминируется Κάτοπτρον при прочтении «инструмент».
    - Ἔλεγχος / Διάγνωσις: именуют метод или половину работы, а не объект; высокий
      клинический / конфронтационный AliasRisk. Это иные governed values, а не худшие синонимы.
    - Ἀνάμνησις: слишком узко (только память); высокий багаж.
    - Φάρμακον: именует парадокс, а не проект; морфология вещества не подходит; высокий
      AliasRisk «яда».
    - Δίοπτρα: «смотреть сквозь», а не «отражать себя» — семантический сдвиг от самоосмотра.
  SelectionRationale:
    Покупает — высшую SemanticFidelity (зеркало = самоосмотр; катоптрика даёт расширяемую
    научную рамку для новых симптомов / оптики / проверок кривизны); наименьший AliasRisk
    на фронте; высшую CognitiveErgonomics; преемственность родословной с итерацией 0 (без
    переименовательной чехарды). Достигнуто здесь независимо по четырём измерениям F.18 —
    не унаследовано.
    Остаточный риск — MorphologicalActionFit высок лишь при прочтении governed value как
    *инструмента*. Если центр тяжести проекта сместится от рефлексии к активному
    исправлению как основному режиму, имя в форме деятельности/метода (Ἔλεγχος) подойдёт
    лучше и front откроется заново.
  BridgeRefs:        AION_AUTOPOIESEON (самосозидание); MNEMOSYNE_PERISTASEOS (память
                     обстоятельств) — см. экосистемные связи в [[choosing_name]].
  UnifiedTermRowRef: нет — локально для этого пространства; F.17 не задействован (F.18:4.4).
  LineageEntries:
    - iter-0 (choosing_name.md): отчеканен Κάτοπτρον через A.7 / A.6.B / A.15.
    - iter-2 (этот файл): независимый прогон F.18; подтверждает Κάτοπτρον по
      SemanticFidelity, CognitiveErgonomics, MorphologicalActionFit (прочтение «инструмент»),
      AliasRisk.
  RefreshCondition:
    Пересмотреть, когда (a) основной режим проекта станет активным исправлением, а не
    рефлексией; (b) изменится управляющий паттерн governed value; или (c) повторяющаяся
    ошибка читателей станет трактовать «зеркало» как «пассивное / без исправлений».
```

## Что этот прогон добавляет к итерации 0

Два инструмента FPF, применённые независимо, сходятся на одном имени — это сильнейшее
подтверждение, какое F.18 может дать уже используемому имени. Сверх подтверждения прогон
вносит три вещи, которые итерация 0 оставила неявными:

1. **Видимый набор кандидатов с правилом доминирования** — итерация 0 сравнивала четыре
   кандидата прозой; F.18 делает front и недоминируемое рассуждение явными и аудируемыми.
2. **AliasRisk как полноправную ось** — она вскрывает конкретный демерит (Κατοπτρισμός =
   «мираж»), который прозовое сравнение не назвало.
3. **RefreshCondition** — напряжение «инструмент против Work», которое итерация 0 пометила
   как «продуктивную слабость», теперь стало операционным триггером для повторного открытия
   имени, а не риторической ремаркой.
